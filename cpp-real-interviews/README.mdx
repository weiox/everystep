# C++ 面試真題 🎙️

> 这里是 C++ 求职者们的“情报中心”！我们收集并整理了来自各大厂的真实 C++ 面试题，希望能为你提供最前沿、最实用的备战资料。

## 🤔 为什么要做这个？

理论知识学得再好，终究要上“战场”检验。我们发现，很多同学在面试中遇到的问题，往往比书本上的更具体、更综合、也更“刁钻”。

这个专栏的目的，就是为了：

- **缩短理论与实践的距离**：让你看看工业界真正在关心什么。
- **提供高效的复习指引**：通过真实题目，查漏补缺，精准打击知识盲区。
- **汇集社区力量**：欢迎大家分享自己的面试经历，众人拾柴火焰高！

## 🚀 如何贡献？

如果你刚刚结束一场 C++ 面试，或者有朋友分享了题目，非常欢迎你通过 Pull Request 的方式，将这些宝贵的“情报”分享出来！

你可以参考以下格式创建一个新的 `.mdx` 文件：

- **文件名**：`公司-岗位-YYYYMMDD[-序号].mdx`
  - 为了有效避免重复，我们采用一个更精确的命名方式。
  - **示例**: `tencent-backend-20240520.mdx`。
  - **关于序号**: 如果你在同一天面试了同一家公司的同一个岗位（例如上午一面，下午二面），或者有多位贡献者在同一天分享了同一家公司、同一岗位的面经，可以在文件名末尾加上序号来区分，例如 `tencent-backend-20240520-2.mdx`。
- **内容模板**：

  ```markdown
  # [公司] [年份] [岗位] 面经

  - **面试轮次**：(例如: 一面, 二面, HR面)
  - **面试形式**：(例如: 电话面试, 视频面试, 现场面试)
  - **面试时长**：(例如: 约 60 分钟)

  ---

  ## 面试问题

  1.  **问题一**：这里是面试官问的具体问题。
      <details>
      <summary>💡 点击查看参考思路</summary>

      这里可以写下你的回答、思考过程，或者相关的知识点链接。

      </details>

  2.  **问题二**：...

  ...

  ## 总结

  这里可以写一些你的面试感想、给后辈的建议，或者对这家公司/这个岗位的看法。
  ```
